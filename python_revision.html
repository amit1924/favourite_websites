Python Basics

Summary

Learn Python fundamentals: variables, data types, operators, and basic I/O operations.

Deep Theory

Python is a dynamically-typed, interpreted language with strong emphasis on readability. Variables are references to objects in memory, and Python uses automatic memory management via reference counting and garbage collection.

Data Types

  • Numeric Types: int, float, complex
  • Sequence Types: str, list, tuple, range
  • Mapping Type: dict
  • Set Types: set, frozenset
  • Boolean Type: bool (True/False)
  • Binary Types: bytes, bytearray, memoryview

Type Conversion

Python provides built-in functions for type conversion: int(), float(), str(), bool(), list(), tuple(), set(), dict().

Control Flow

Summary

Master conditional statements, loops, and flow control mechanisms in Python.

Deep Theory

Control flow statements determine the order in which code executes. Python provides:

Conditional Statements

  • if, elif, else for conditional execution
  • Python uses indentation to define code blocks (typically 4 spaces)
  • Conditions evaluate to True or False
  • Python supports chained comparisons: a < b < c

Loops

  • for loop: iterates over sequences (lists, strings, ranges, etc.)
  • while loop: repeats while a condition is true
  • Loop control: break (exit loop), continue (skip to next iteration)
  • else clause in loops: executes when loop completes normally (no break)

Problem: Write a program that prints the Fibonacci sequence up to a given number n.

Fibonacci sequence: Each number is the sum of the two preceding ones (0, 1, 1, 2, 3, 5, 8, ...)

Time estimate: 10-15 minutes

Solution:


def fibonacci_sequence(n):
    """Generate Fibonacci sequence up to n"""
    sequence = []
    a, b = 0, 1
    while a <= n:
        sequence.append(a)
        a, b = b, a + b
    return sequence

# Test the function
limit = 50
fib_seq = fibonacci_sequence(limit)
print(f"Fibonacci sequence up to {limit}: {fib_seq}")

Collections

Summary

Master Python's built-in collection types: lists, tuples, sets, and dictionaries.

Deep Theory

List

  • Mutable, ordered sequence of elements
  • Time complexity: O(1) for append/pop from end, O(n) for insert/delete from beginning
  • Supports slicing, list comprehensions

Tuple

  • Immutable, ordered sequence
  • Faster than lists for iteration
  • Can be used as dictionary keys (if all elements are hashable)

Set

  • Unordered collection of unique, hashable elements
  • Time complexity: O(1) for membership test on average
  • Supports mathematical set operations (union, intersection, difference)

Dictionary

  • Key-value pairs, keys must be hashable
  • Time complexity: O(1) for lookup, insert, delete on average
  • Implemented as hash tables
  • Python 3.7+ maintains insertion order

Practice Exercise

Problem: Write a function that counts the frequency of each word in a given text and returns the top N most frequent words.

Time estimate: 15-20 minutes

Solution:


def top_n_words(text, n=5):
    """Return the top N most frequent words in the text"""
    # Convert to lowercase and split into words
    words = text.lower().split()
    
    # Count word frequencies
    word_count = {}
    for word in words:
        # Remove punctuation (basic approach)
        word = word.strip('.,!?;:"')
        if word:
            word_count[word] = word_count.get(word, 0) + 1
    
    # Sort by frequency (descending) and get top N
    sorted_words = sorted(word_count.items(), key=lambda x: x[1], reverse=True)
    return sorted_words[:n]

# Test the function
sample_text = "Hello world! Hello Python. Python is great. World of Python."
top_words = top_n_words(sample_text, 3)
print("Top words:")
for word, count in top_words:
    print(f"  {word}: {count}")

Functions

Summary

Learn to create reusable code with functions, understand parameters, return values, and advanced function concepts.

Deep Theory

Function Definition

  • Use def keyword to define functions
  • Functions are first-class objects in Python
  • Can be assigned to variables, passed as arguments, returned from other functions

Parameters

  • Positional arguments: Passed in order
  • Keyword arguments: Passed with parameter names
  • Default parameters: Parameters with default values
  • *args: Variable number of positional arguments
  • **kwargs: Variable number of keyword arguments

Advanced Concepts

  • Closures: Functions that remember values in enclosing scope
  • Decorators: Functions that modify other functions
  • Lambda functions: Anonymous functions
  • Recursion: Functions that call themselves

Practice Exercise

Problem: Create a decorator that caches function results to avoid recomputation for the same inputs.

Time estimate: 15-20 minutes

Solution:


def cache_decorator(func):
    """Decorator to cache function results"""
    cache = {}
    
    def wrapper(*args):
        # Use args as key (must be hashable)
        if args in cache:
            print(f"Cache hit for {args}")
            return cache[args]
        else:
            print(f"Computing result for {args}")
            result = func(*args)
            cache[args] = result
            return result
    
    return wrapper

@cache_decorator
def fibonacci(n):
    """Compute nth Fibonacci number"""
    if n < 2:
        return n
    return fibonacci(n-1) + fibonacci(n-2)

# Test the cached Fibonacci function
print(f"fibonacci(5) = {fibonacci(5)}")
print(f"fibonacci(10) = {fibonacci(10)}")
print(f"fibonacci(5) = {fibonacci(5)}")  # This should use cache

Object-Oriented Programming

Summary

Master Python's object-oriented programming concepts: classes, inheritance, polymorphism, and design patterns.

Deep Theory

Classes and Objects

  • Class: Blueprint for creating objects
  • Object: Instance of a class
  • Constructor: __init__ method
  • Instance variables: Data unique to each instance
  • Class variables: Shared across all instances

Inheritance

  • Single inheritance: Class inherits from one parent
  • Multiple inheritance: Class inherits from multiple parents
  • Method Resolution Order (MRO): Determines method lookup order
  • super(): Call parent class methods

Polymorphism & Encapsulation

  • Polymorphism: Same interface, different implementations
  • Encapsulation: Bundling data with methods
  • Abstraction: Hiding implementation details

Special Methods

  • __str__: String representation
  • __repr__: Official string representation
  • __eq__, __hash__: Equality and hashing
  • __getitem__, __setitem__: Indexing support

Real-world Examples 


# Basic class example
class Person:
    """A simple Person class"""
    
    # Class variable (shared by all instances)
    species = "Homo sapiens"
    
    def __init__(self, name, age):
        # Instance variables (unique to each instance)
        self.name = name
        self.age = age
    
    def greet(self):
        return f"Hello, my name is {self.name} and I'm {self.age} years old"
    
    def __str__(self):
        return f"Person(name='{self.name}', age={self.age})"
    
    def __repr__(self):
        return f"Person('{self.name}', {self.age})"

# Inheritance example
class Student(Person):
    """Student class inheriting from Person"""
    
    def __init__(self, name, age, student_id, major):
        # Call parent class constructor
        super().__init__(name, age)
        self.student_id = student_id
        self.major = major
    
    def study(self, subject):
        return f"{self.name} is studying {subject}"
    
    def greet(self):  # Override parent method
        return f"Hi, I'm {self.name}, a {self.major} student"

# Using the classes
person = Person("Alice", 25)
student = Student("Bob", 20, "S12345", "Computer Science")

print(person.greet())
print(student.greet())
print(student.study("Python Programming"))

print(f"Person species: {Person.species}")
print(f"Student species: {Student.species}")

def introduce(person):

class Circle:
    """Circle class with property for diameter"""
    
    def __init__(self, radius):
        self._radius = radius
    
    @property
    def radius(self):
        return self._radius
    
    @radius.setter
    def radius(self, value):
        if value < 0:

        self._radius = value

    def diameter(self):

        self.radius = value / 2

    def area(self):

Practice Exercise

Problem: Create a banking system with Account classes (SavingsAccount, CheckingAccount) that demonstrate inheritance, polymorphism, and encapsulation.

Time estimate: 25-30 minutes

Solution:


from abc import ABC, abstractmethod
from datetime import datetime

class Account(ABC):
    """Abstract base class for bank accounts"""
    
    def __init__(self, account_number, owner, balance=0):
        self.account_number = account_number
        self.owner = owner
        self._balance = balance
        self.transactions = []
    
    @property
    def balance(self):
        return self._balance
    
    def deposit(self, amount):
        if amount <= 0:
            raise ValueError("Deposit amount must be positive")
        self._balance += amount
        self.transactions.append(Transaction("DEPOSIT", amount, datetime.now()))
        return self._balance
    
    @abstractmethod
    def withdraw(self, amount):
        pass
    
    def __str__(self):
        return f"{self.__class__.__name__}({self.account_number}, {self.owner}, balance=${self.balance:.2f})"

class SavingsAccount(Account):
    """Savings account with interest"""
    
    def __init__(self, account_number, owner, balance=0, interest_rate=0.01):
        super().__init__(account_number, owner, balance)
        self.interest_rate = interest_rate
    
    def withdraw(self, amount):
        if amount <= 0:
            raise ValueError("Withdrawal amount must be positive")
        if amount > self._balance:
            raise ValueError("Insufficient funds")
        self._balance -= amount
        self.transactions.append(Transaction("WITHDRAWAL", -amount, datetime.now()))
        return self._balance
    
    def add_interest(self):
        interest = self._balance * self.interest_rate
        self._balance += interest
        self.transactions.append(Transaction("INTEREST", interest, datetime.now()))
        return interest

class CheckingAccount(Account):
    """Checking account with overdraft protection"""
    
    def __init__(self, account_number, owner, balance=0, overdraft_limit=100):
        super().__init__(account_number, owner, balance)
        self.overdraft_limit = overdraft_limit
    
    def withdraw(self, amount):
        if amount <= 0:
            raise ValueError("Withdrawal amount must be positive")
        if amount > (self._balance + self.overdraft_limit):
            raise ValueError("Exceeds overdraft limit")
        self._balance -= amount
        self.transactions.append(Transaction("WITHDRAWAL", -amount, datetime.now()))
        return self._balance

class Transaction:
    """Transaction record"""
    
    def __init__(self, type, amount, timestamp):
        self.type = type
        self.amount = amount
        self.timestamp = timestamp
    
    def __str__(self):
        return f"{self.timestamp.strftime('%Y-%m-%d %H:%M')} {self.type}: ${self.amount:+.2f}"

# Test the banking system
savings = SavingsAccount("SAV001", "Alice", 1000, 0.02)
checking = CheckingAccount("CHK001", "Bob", 500, 200)

print(savings)
print(checking)

savings.deposit(200)
savings.withdraw(100)
interest = savings.add_interest()

checking.withdraw(600)  # Within overdraft limit

print(f"Savings balance after transactions: ${savings.balance:.2f}")
print(f"Checking balance after withdrawal: ${checking.balance:.2f}")

print("\nSavings transactions:")
for t in savings.transactions:
    print(f"  {t}")

def is_palindrome(s):
    # Remove non-alphanumeric characters and convert to lowercase
    cleaned = re.sub(r'[^a-zA-Z0-9]', '', s).lower()
    return cleaned == cleaned[::-1]

3. Two Sum

Medium

Description: Given an array of integers and a target sum, return indices of the two numbers that add up to the target. Assume exactly one solution exists.

Time estimate: 15-20 minutes


def two_sum(nums, target):
    num_map = {}
    for i, num in enumerate(nums):
        complement = target - num
        if complement in num_map:
            return [num_map[complement], i]
        num_map[num] = i
    return []

# Test
nums = [2, 7, 11, 15]
target = 9
print(two_sum(nums, target))  # [0, 1]

# Complexity Analysis:
# Time: O(n) - single pass through the array
# Space: O(n) - storing numbers in hash map

4. Binary Search Tree Validation

Medium

Description: Given the root of a binary tree, determine if it is a valid binary search tree (BST).

Time estimate: 20-25 minutes


class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

def is_valid_bst(root):
    def validate(node, low=float('-inf'), high=float('inf')):
        if not node:
            return True
        
        if node.val <= low or node.val >= high:
            return False
        
        return (validate(node.left, low, node.val) and 
                validate(node.right, node.val, high))
    
    return validate(root)

# Test
#     2
#    / \
#   1   3
root = TreeNode(2)
root.left = TreeNode(1)
root.right = TreeNode(3)
print(is_valid_bst(root))  # True

#     5
#    / \
#   1   4
#      / \
#     3   6
root2 = TreeNode(5)
root2.left = TreeNode(1)
root2.right = TreeNode(4)
root2.right.left = TreeNode(3)
root2.right.right = TreeNode(6)
print(is_valid_bst(root2))  # False

# Complexity Analysis:
# Time: O(n) - visit each node once
# Space: O(h) - recursion stack, where h is height of tree

5. LRU Cache Implementation

Hard

Description: Design and implement a Least Recently Used (LRU) cache. It should support get and put operations in O(1) time complexity.

Time estimate: 30-40 minutes


class ListNode:
    def __init__(self, key=0, val=0, prev=None, next=None):
        self.key = key
        self.val = val
        self.prev = prev
        self.next = next

class LRUCache:
    def __init__(self, capacity: int):
        self.capacity = capacity
        self.cache = {}
        # Dummy head and tail for the doubly linked list
        self.head = ListNode()
        self.tail = ListNode()
        self.head.next = self.tail
        self.tail.prev = self.head
    
    def _add_node(self, node):
        """Add node right after head"""
        node.prev = self.head
        node.next = self.head.next
        self.head.next.prev = node
        self.head.next = node
    
    def _remove_node(self, node):
        """Remove an existing node from the linked list"""
        prev = node.prev
        next = node.next
        prev.next = next
        next.prev = prev
    
    def _move_to_head(self, node):
        """Move certain node to head (most recently used)"""
        self._remove_node(node)
        self._add_node(node)
    
    def _pop_tail(self):
        """Pop the current tail (least recently used)"""
        res = self.tail.prev
        self._remove_node(res)
        return res

    def get(self, key: int) -> int:
        node = self.cache.get(key, None)
        if not node:
            return -1
        # Move the accessed node to the head
        self._move_to_head(node)
        return node.val

    def put(self, key: int, value: int) -> None:
        node = self.cache.get(key)
        if not node:
            new_node = ListNode(key, value)
            self.cache[key] = new_node
            self._add_node(new_node)
            if len(self.cache) > self.capacity:
                # Pop the tail
                tail = self._pop_tail()
                del self.cache[tail.key]
        else:
            # Update the value
            node.val = value
            self._move_to_head(node)

# Test
cache = LRUCache(2)
cache.put(1, 1)
cache.put(2, 2)
print(cache.get(1))  # 1
cache.put(3, 3)      # evicts key 2
print(cache.get(2))  # -1
cache.put(4, 4)      # evicts key 1
print(cache.get(1))  # -1
print(cache.get(3))  # 3
print(cache.get(4))  # 4

# Complexity Analysis:
# Time: O(1) for both get and put operations
# Space: O(capacity) - for the hash map and linked list

# Filter list
evens = list(filter(lambda x: x % 2 == 0, range(10)))

# Map function
doubled = list(map(lambda x: x * 2, [1, 2, 3]))

# Swap variables
a, b = b, a

# Flatten list of lists
flat_list = [item for sublist in list_of_lists for item in sublist]

# Check if all elements satisfy condition
all_positive = all(x > 0 for x in [1, 2, 3])

# Check if any element satisfies condition
has_negative = any(x < 0 for x in [1, -2, 3])

Common Pitfalls

Mutable Default Arguments

Don't use mutable objects as default arguments: 


# Wrong
def append_to(element, target=[]):
    target.append(element)
    return target

# Correct
def append_to(element, target=None):
    if target is None:
        target = []
    target.append(element)
    return target

Variable Scope in Comprehensions

List comprehensions have their own scope in Python 3+: 


x = 10
# This works fine in Python 3
squares = [x**2 for x in range(5)]
# x is still 10 here

Copying Lists

Use copy() or slicing to create new lists: 


original = [1, 2, 3]
# Wrong - both reference same list
copy_wrong = original
# Correct - creates new list
copy_correct = original.copy()
# Also correct
copy_correct2 = original[:]
